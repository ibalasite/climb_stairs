API Reference — Ladder Room Online

Version v1.0 — 2026-04-19 — Based on EDD v1.1 + PRD v1.1

1. 概覽

Base URL

Production:  https://api.ladder-room.online/api/v1
Development: http://localhost:3000/api/v1
WebSocket:   wss://api.ladder-room.online/ws
  • 所有 HTTP REST 端點掛載於 /api/v1/。版本號嵌入路徑（Path Versioning）。
  • WebSocket 不含版本路徑；由 query param token 的 JWT iat 隱含版本邊界。
  • 所有請求與回應均使用 Content-Type: application/json; charset=utf-8。

認證

Host 專屬操作需在 Authorization Header 帶入 JWT：

Authorization: Bearer <sessionToken>

sessionToken 由 POST /api/v1/rooms（建立房間）或 POST /api/v1/rooms/:code/players（加入房間）取得。

統一回應 Envelope

interface ApiResponse<T> {
  success: boolean;
  data: T | null;
  error: { code: string; message: string } | null;
}

POST /api/v1/rooms

建立新房間。回傳 6 碼 Room Code 及 host sessionToken。

Request Body

{ hostNickname: string; winnerCount?: number }

Response 201

{
  "success": true,
  "data": {
    "roomCode": "AB3K7X",
    "playerId": "uuid-host-001",
    "sessionToken": "eyJ..."
  },
  "error": null
}
GET /api/v1/rooms/:code

查詢房間摘要，包含狀態、玩家數、是否可加入。不返回 ladder 結構。

Response 200

{
  "success": true,
  "data": {
    "code": "AB3K7X",
    "status": "waiting",
    "playerCount": 3,
    "maxPlayers": 50,
    "canJoin": true
  },
  "error": null
}
POST /api/v1/rooms/:code/players

玩家加入房間。回傳 playerId 及 sessionToken（用於 WebSocket 連線）。

Request Body

{ nickname: string }

Response 201

{
  "success": true,
  "data": {
    "playerId": "uuid-player-002",
    "sessionToken": "eyJ...",
    "colorIndex": 2
  },
  "error": null
}
DELETE /api/v1/rooms/:code/players/:id Host JWT

主持人踢除指定玩家。只在 waiting 狀態下允許。

Response 200

{ "success": true, "data": { "kickedPlayerId": "uuid-player-002" }, "error": null }
POST /api/v1/rooms/:code/start Host JWT

主持人開始遊戲。驗證 N≥2 且 W 已設定，生成 LadderData 並廣播 ROOM_STATE。

Request Body

{ winnerCount: number }

Response 200

{
  "success": true,
  "data": {
    "status": "running",
    "ladder": { "seed": 3141592653, "rowCount": 30, "colCount": 10 }
  },
  "error": null
}
POST /api/v1/rooms/:code/reset Host JWT

再玩一局。剔除離線玩家，清空 kickedPlayerIds，重置狀態為 waiting。

3. WebSocket

3.1 連線握手

wss://api.ladder-room.online/ws?token=<sessionToken>&room=<roomCode>

伺服器驗證 JWT HMAC-SHA256 簽章，失敗時關閉連線（code 4001）。成功後立即發送 ROOM_STATE_FULL。

3.2 Server → Client Events

ROOM_STATE_FULL

完整房間狀態，連線後及任何重要狀態變更後發送。含 players、ladder（null 若未開始）。

ROOM_STATE

部分更新廣播（玩家加入/離線/W 變更等）。payload 僅含變更欄位。

REVEAL_INDEX

揭曉指定路徑。payload: { index: number, playerId: string }

REVEAL_ALL

一鍵揭曉所有剩餘路徑。payload: { results: ResultSlot[] }

PLAYER_KICKED

玩家被踢除。payload: { playerId: string }。被踢者收到此訊息後應跳轉至錯誤頁面。

ERROR

業務邏輯錯誤。payload: { code: string, message: string }

3.3 Client → Server Messages

Type權限Payload說明
SET_WINNER_COUNTHost{ winnerCount: number }設定中獎名額
START_GAMEHost{}開始遊戲（需先呼叫 /start HTTP）
