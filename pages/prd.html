PRD — Ladder Room Online

Version v1.1 — 2026-04-19 — Based on PDD v2.1

執行摘要

Ladder Room Online 是一款基於 HTML5 Canvas 的多人線上爬樓梯抽獎遊戲，支援最多 50 人同房間即時參與。主持人建立房間並掌控揭曉節奏，玩家透過 6 碼房間碼加入並即時觀看路徑動畫；Mulberry32 PRNG 算法確保每局結果在所有客戶端 100% 一致，消除舞弊疑慮，適用於團隊抽獎、課堂點名、活動互動等場景。

主持人（Host）User Stories

US-H01 建立帶有唯一 6 碼房間碼的房間

作為主持人，我想要建立一個帶有唯一 6 碼房間碼的房間，以便玩家能快速找到並加入我的抽獎活動。

AC-H01-1 Given 主持人點擊「建立房間」，When 系統接受請求，Then 於 2 秒內回傳 6 碼 Room Code（字元集 [A-HJ-NP-Z2-9]，長度固定 6 碼）且在 Redis 中不重複。
AC-H01-2 Given 房間建立成功，When 主持人進入房間，Then 頁面顯示 Room Code、目前玩家列表（空）、設定中獎名額的輸入框，狀態為 waiting。
AC-H01-3 Given 建立房間的請求因網路異常失敗，When 系統偵測到錯誤，Then 顯示「建立失敗，請重試」提示，不產生孤立房間。
US-H02 設定中獎名額

作為主持人，我想要設定中獎名額（W），以便控制本局有多少玩家會獲獎。

AC-H02-1 Given 房間狀態為 waiting，When 主持人填入 W（1 ≤ W ≤ N-1），Then 系統接受並廣播更新後的 ROOM_STATE。
AC-H02-2 Given 主持人輸入 W ≤ 0 或 W ≥ N，When 送出，Then 系統顯示「中獎名額須介於 1 到玩家數減 1 之間」錯誤。
AC-H02-3 Given 再玩一局後 W ≥ 新 N，When 系統偵測到越界，Then 自動將 W 重設為 null 並通知主持人。
US-H03 開始遊戲

作為主持人，我想要在所有玩家就緒後開始遊戲，以便正式進入揭曉流程。

AC-H03-1 Given N ≥ 2 且 W 已設定，When 主持人點擊「開始遊戲」，Then 狀態轉為 running，後端生成樓梯並廣播 ROOM_STATE。
AC-H03-2 Given W 尚未設定，When 主持人點擊開始，Then 拒絕並回傳「請先設定中獎名額」。
AC-H03-4 Given N < 2，When 主持人點擊開始，Then 系統回傳 INSUFFICIENT_PLAYERS，顯示「人數不足（至少需要 2 位玩家）」。
AC-H03-5 rowCount = clamp(N×3, 20, 60)。範例：N=3→20；N=10→30；N=21→60。
US-H04 / H05 / H06 揭曉控制

手動逐步：主持人每次點擊「下一位」廣播 REVEAL_INDEX，所有端同步播放動畫。

自動揭曉：主持人設定 1-30 秒間隔，伺服器自動廣播，可隨時切回手動。

一鍵全部：廣播 REVEAL_ALL，所有路徑壓縮播放，2 秒內完成渲染，狀態自動轉 finished。

US-H07 踢除玩家
AC-H07-1 waiting 狀態下踢除 → 廣播 PLAYER_KICKED，被踢玩家跳轉至「你已被移出房間」頁面。
AC-H07-2 被踢玩家以相同 playerId 重連 → 伺服器拒絕，回傳 PLAYER_KICKED。
AC-H07-3 running 或之後狀態踢除 → 拒絕，回傳 INVALID_STATE。
AC-H07-5 再玩一局後 kickedPlayerIds 清空，前一局被踢者可用新暱稱加入新局。
US-H08 再玩一局
AC-H08-1 finished 狀態下，主持人點擊「再玩一局」→ 剔除 isOnline=false 玩家，重置為 waiting，廣播新 ROOM_STATE。
AC-H08-3 在線玩家數 < 2 時，系統拒絕並提示「在線玩家不足，無法開始新局」。

玩家（Player）User Stories

US-P01 加入房間
AC-P01-1 玩家輸入有效暱稱（1-20 Unicode 字元）及正確 Room Code → 1.5 秒內進入等待畫面。
AC-P01-2 暱稱重複 → 回傳 NICKNAME_TAKEN，提示「此暱稱已被使用，請換一個」。
AC-P01-4 房間已達 50 人 → 回傳 ROOM_FULL，顯示「房間已滿，無法加入」。
US-P02 / P03 / P04 遊戲體驗

即時玩家列表：新玩家加入或離線時，所有端 2 秒內更新。離線玩家顯示「離線」標記，不立即移除。

路徑動畫：輪到自己的路徑高亮播放；其他玩家路徑以淡色呈現。桌機 ≥ 30fps，手機 ≥ 24fps。

結果顯示：自己的路徑抵達底部時，顯示「恭喜中獎！」或「未中獎」，結果與伺服器端一致。
