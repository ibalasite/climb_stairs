Ladder Room Online

A real-time, room-based 爬樓梯 (ladder climbing) lottery game supporting up to 50 players. Built with Node.js, TypeScript, Fastify, WebSockets, and HTML5 Canvas — zero UI framework on the client.

Technology Stack

Node.js 20 TypeScript (strict) Fastify ws (WebSocket) Redis Vanilla TypeScript HTML5 Canvas Vite Clean Architecture Monorepo Kubernetes HPA GitHub Pages CDN

The monorepo contains three packages: packages/shared (zero-I/O domain logic shared by client and server), packages/server (Fastify + ws + Redis), and packages/client (Vanilla TS + Vite + Canvas). Clean Architecture ensures the PRNG algorithm is identical on both ends — no result discrepancy is possible.

Core Concepts

Rooms & Lifecycle

A host creates a room and receives a unique 6-character code (A-Z excluding O/I, 2-9, 32-character set). Players join by entering the code or following a share link. The room state machine transitions: waiting → running → revealing → finished, with an optional reset back to waiting.

Deterministic PRNG

Ladder structure is generated using djb2 hashing for the seed, Mulberry32 as the PRNG, and Fisher-Yates shuffle. The same seed always produces the same ladder on every client — making results tamper-proof and verifiable.

Real-Time Sync

All state changes (player joins, game start, reveal steps) are broadcast via WebSocket. Redis Pub/Sub fans out messages across Kubernetes pods. Nginx Ingress uses sticky sessions to keep a room's players on the same pod.
