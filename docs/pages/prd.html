PRD — 產品功能規格

User Stories

主持人（Host）

  • US-H01：身為主持人，我想在 30 秒內建立一個房間並取得分享連結，以便快速開始活動
  • US-H02：身為主持人，我想即時看到誰已加入等待室，以便確認人數
  • US-H03：身為主持人，我想設定中獎名額後一鍵開始遊戲，以便控制活動節奏
  • US-H04：身為主持人，我想手動或自動逐一揭曉走線結果，以便製造懸念
  • US-H05：身為主持人，我想在任何時候一鍵全部揭曉，以便快速結束活動
  • US-H06：身為主持人，我想查看完整得獎名單，以便公告結果
  • US-H07：身為主持人，我想踢除特定玩家，以便管理房間秩序
  • US-H08：身為主持人，我想在本局結束後一鍵開始下一局，以便連續舉辦多場抽獎

玩家（Player）

  • US-P01：身為玩家，我想透過連結或 6 碼房間碼加入房間，無需安裝任何 App
  • US-P02：身為玩家，我想在等待室看到其他玩家的暱稱，以便確認自己已成功加入
  • US-P03：身為玩家，我想在遊戲開始時看到梯子動畫揭曉，以便感受抽獎過程的緊張感
  • US-P04：身為玩家，我想在結果頁面清楚看到自己是否中獎，以及完整走線路徑
  • US-P05：身為玩家，我想在斷線後重連時自動恢復到正確的遊戲畫面

功能需求（FR）

FR-01 房間生命週期

  • FR-01-1：系統應在收到 POST /api/rooms 後 1 秒內建立房間並回傳 roomCode、playerId、sessionToken
  • FR-01-2：房間在 waiting/running 狀態下若 4 小時無任何活動，自動過期（Redis TTL）
  • FR-01-3：所有玩家斷線超過 5 分鐘後，房間自動清除
  • FR-01-4：finished 狀態房間 TTL 延長至 1 小時（供玩家截圖存證）

FR-02 玩家加入

  • FR-02-1：玩家輸入有效暱稱（1-20 Unicode 碼位）後可加入 waiting 狀態房間
  • FR-02-2：同一房間中暱稱不可重複（含 isOnline=false 的離線玩家）
  • FR-02-3：房間人數上限 50 人；超過時回傳 ROOM_FULL
  • FR-02-4：running / revealing / finished 狀態下不允許新玩家加入

FR-03 遊戲開始

  • FR-03-1：Host 點擊開始遊戲前，Server 重新驗證 N ≥ 2 且 1 ≤ W ≤ N-1
  • FR-03-2：驗證通過後，Server 設定 status=running 並廣播 ROOM_STATE
  • FR-03-3：Server 於 running 狀態執行原子操作：生成 seed、ladderMap、resultSlots，設定 status=revealing，廣播 ROOM_STATE_FULL
  • FR-03-4：原子操作若 10 秒逾時，Server 回退至 waiting 並廣播帶 reason 的 ROOM_STATE

FR-04 梯子生成

  • FR-04-1：rowCount = clamp(N×3, 20, 60)，colCount = N
  • FR-04-2：每列橫槓數 = max(1, round(N/4))；橫槓不重疊、不相鄰
  • FR-04-3：startColumn 指派：players 依 joinedAt 升序排列，以 Mulberry32 PRNG 洗牌後依序指派
  • FR-04-4：相同 seed 必產生相同梯子結構（確定性）

FR-05 揭曉流程

  • FR-05-1：REVEAL_NEXT 由 Host 手動觸發，Server 以原子 INCR 遞增 revealedCount，廣播 REVEAL_INDEX
  • FR-05-2：自動揭曉模式：Host 設定 intervalSec（1-30s），Server 定時廣播 REVEAL_INDEX
  • FR-05-3：REVEAL_ALL_TRIGGER：Server 設定 revealedCount = N，廣播 REVEAL_ALL（path 省略，64KB 限制）
  • FR-05-4：所有路徑揭曉後，Host 可觸發 END_GAME；Server 設定 status=finished，廣播 ROOM_STATE（含 seed）

FR-06 斷線重連

  • FR-06-1：玩家帶有效 playerId 重連時，Server unicast ROOM_STATE_FULL（完整快照）
  • FR-06-2：Host 斷線後 60s 若未重連，自動移交主持權給下一位在線玩家
  • FR-06-3：同一 playerId 從新裝置連線，舊 session 收到 SESSION_REPLACED 後斷線

FR-07 玩家踢除

  • FR-07-1：Host 在 waiting 狀態下可踢除任意非 Host 玩家
  • FR-07-2：Server 推送 PLAYER_KICKED 給被踢玩家（若仍在線），並強制關閉其 WS 連線
  • FR-07-3：被踢玩家 playerId 加入 kickedPlayerIds；禁令於 Host 點擊「再玩一局」後解除

FR-08 再玩一局

  • FR-08-1：Host 在 finished 狀態下點擊「再玩一局」，Server pruning isOnline=false 玩家
  • FR-08-2：清除 kickedPlayerIds、重置遊戲欄位（seed、ladder、results、revealedCount）
  • FR-08-3：設定 status=waiting 並廣播 ROOM_STATE

非功能需求（NFR）

編號類型需求
NFR-01效能50 並發 WS 連線，P95 訊息延遲 < 500ms（一般 WiFi）
NFR-02效能梯子揭曉動畫：桌機 ≥ 30fps，手機 ≥ 24fps（近 4 年 mid-range 裝置）
NFR-03可用性建立房間成功率 > 99.5%；玩家加入成功率 > 99%
NFR-04安全性JWT HS256 認證主持人操作；JWT TTL 6 小時
NFR-05安全性seed 在 finished 前不對任何客戶端公開
NFR-06安全性WS maxPayload = 65536 bytes；Rate limiting 於所有 HTTP 端點
NFR-07相容性Chrome / Firefox / Safari 近 2 年版本；iOS / Android 手機瀏覽器
NFR-08響應式最小支援寬度 320px；N > 22 時 Canvas 橫向捲動
NFR-09Bundle前端 JS < 150KB gzip；無 UI 框架依賴
NFR-10一致性前後端共用 PRNG 程式碼（packages/shared），相同 seed 必產生相同結果

驗收條件（AC）

編號情境條件
AC-H01-1建立房間Host 填寫暱稱後點擊「建立房間」，1 秒內進入等待大廳，頁面顯示 6 碼 Room Code
AC-H01-2分享連結點擊「複製連結」後，clipboard 包含正確的 Join URL
AC-H02-1玩家加入通知玩家加入後，Host 等待大廳在 1 秒內即時更新玩家列表（無刷新）
AC-H03-1開始遊戲N ≥ 2 且 W 設定合法後，Host 點擊「開始遊戲」，所有客戶端在 2 秒內切換至遊戲畫面
AC-H03-2開始驗證N < 2 時顯示「人數不足（至少需要 2 位玩家）」；W 未設定時顯示「請先設定中獎名額」
AC-H04-1手動揭曉Host 點擊「下一位」，目標玩家走線動畫在 2 秒內開始播放
AC-H04-2自動揭曉切換至自動模式後，Server 依設定間隔自動廣播 REVEAL_INDEX，誤差 ≤ 500ms
AC-H05-1一鍵全揭Host 點擊「全部揭曉」，2 秒內所有路徑靜態顯示，REVEAL_ALL 廣播 payload ≤ 64KB
AC-H06-1結果頁面finished 後，Host 看到完整得獎名單（暱稱 + 結果 + seed 資訊）
AC-H07-1踢除玩家Host 點擊踢除後，被踢玩家 1 秒內收到 PLAYER_KICKED 並導回首頁
AC-H07-2踢除禁令被踢玩家以相同 playerId 重連，收到 PLAYER_KICKED 錯誤（4003 close code）
AC-H07-3踢除狀態限制running / revealing / finished 狀態下踢除請求回傳 INVALID_STATE
AC-H07-4踢除 Host嘗試踢除 Host 回傳 CANNOT_KICK_HOST
AC-H07-5再玩一局禁令清除Host 點擊「再玩一局」後，前一局被踢玩家可以新暱稱重新加入
AC-H08-1再玩一局Host 點擊「再玩一局」，所有 isOnline=false 玩家從名單移除，房間回 waiting
AC-P01-1玩家加入玩家透過 Join URL 加入，填寫暱稱後在 10 秒內進入等待大廳
AC-P01-2暱稱重複暱稱已存在時，回傳 NICKNAME_TAKEN，客戶端顯示提示並保持在加入頁面
AC-P02-1斷線重連玩家斷線後重連，帶 playerId 重連後 2 秒內恢復正確遊戲畫面
AC-P02-2心跳偵測ws 內建 ping/pong 間隔 30s；30s 無 pong 回應視為斷線
AC-P03-1揭曉動畫REVEAL_INDEX 到達後，走線動畫在 500ms 內開始播放，幀率符合 NFR-02
AC-P04-1結果顯示finished 狀態下，玩家頁面顯示自己的結果（中獎 / 未中獎）及完整走線路徑
AC-SYS-01PRNG 一致性以相同 seed 在前後端分別執行 generateLadder，結果 100% 一致
AC-SYS-02原子性running→revealing 原子操作在 Redis WATCH/MULTI/EXEC 中完成；失敗時不留部分資料
AC-SYS-03健康檢查GET /health 回傳 200 含 redis、wsCount、uptime；redis=error 時 GET /ready 回傳 503

Out of Scope（MVP 範圍外）

  1. 自訂獎項名稱（目前僅 win / lose，無獎項文字）
  2. 觀眾模式（無 playerId 的唯讀觀看者）
  3. 多房間儀表板
  4. 主持人移交功能（HOST_TRANSFERRED 事件保留設計，Post-MVP 實作）
  5. WebSocket perMessageDeflate 壓縮（REVEAL_ALL path 省略作為 MVP 暫解）
  6. 玩家頭像 / 圖片上傳
  7. 歷史紀錄查詢
  8. 多語言 i18n（僅繁體中文）
