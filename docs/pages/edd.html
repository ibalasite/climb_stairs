1. 系統概覽

Ladder Room Online 是一款基於 HTML5 Canvas 的多人線上爬樓梯抽獎系統，採用 WebSocket 長連接驅動即時遊戲狀態同步，支援最多 50 名玩家共享同一房間。後端以 Fastify 處理 HTTP REST 操作，ws 原生 WebSocket 處理即時通訊，Redis 同時承擔分散式狀態鎖（原子操作）、房間資料持久化與跨 Pod Pub/Sub 廣播的角色；前端以 Vanilla TypeScript + Vite 建構，透過 HTML5 Canvas 逐段繪製梯子揭示動畫，全程無任何 UI 框架依賴，確保最小 JS bundle。

分層職責

層級職責依賴方向
Domain (shared)Entity、Value Object、純業務規則、DomainError無外部依賴
Use Cases (shared)協調 Domain 物件完成業務流程，返回純資料結構僅依賴 Domain
Application (server)呼叫 Use Cases、協調 Repository、發布 WS 事件依賴 Use Cases + Interfaces
Infrastructure (server)Redis 實作、WebSocket 封裝、外部 SDK實作 Application 定義的 Interface
Presentation (server)HTTP Route、Schema 驗證、WS 訊息分派依賴 Application Service

DI 策略： constructor injection，container.ts 以工廠函式組裝所有依賴。測試時直接傳入 mock 實作，不需要 DI 框架即可達成可測試性。

2.3 資料流圖（加入房間流程）

sequenceDiagram
    participant C as Client Browser
    participant N as Nginx Ingress
    participant F as Fastify Pod
    participant R as Redis

    C->>N: POST /api/v1/rooms/join
    N->>F: HTTP forward sticky session
    F->>R: SETNX room:{code} atomic
    R-->>F: OK / room data
    F->>R: SADD room:{code}:players {playerId}
    R-->>F: OK
    F-->>C: 201 { roomCode, playerId, sessionToken }

    C->>N: GET WSS /ws?room={code}&token={token}
    N->>F: WebSocket Upgrade sticky session
    F->>F: Verify sessionToken HMAC-SHA256
    F->>R: HGET room:{code} load state
    R-->>F: Room JSON
    F-->>C: WS Connected
    F->>C: ROOM_STATE_FULL { room, players, ladder:null }

    Note over C,F: Host triggers START_GAME

    C->>F: WS MSG: START_GAME
    F->>R: WATCH room:{code} MULTI SET status=running EXEC
    R-->>F: EXEC OK
    F->>R: PUBLISH room:{code}:events ROOM_STATE
    F-->>C: Broadcast ROOM_STATE { status: running, rowCount }

    Note over C,F: Host triggers BEGIN_REVEAL

    C->>F: WS MSG: BEGIN_REVEAL
    F->>R: SET status=revealing + ladderMap + resultSlots atomic
    R-->>F: OK
    F->>R: PUBLISH room:{code}:events ROOM_STATE_FULL
    F-->>C: Broadcast ROOM_STATE_FULL { status: revealing, ladder }

    Note over C,F: Host triggers REVEAL_NEXT

    C->>F: WS MSG: REVEAL_NEXT {}
    F->>R: INCR room:{code}:revealedCount atomic
    R-->>F: newCount
    F->>R: PUBLISH room:{code}:events REVEAL_INDEX
    F-->>C: Broadcast REVEAL_INDEX { playerIndex, path, result }

    Note over C,F: Host triggers END_GAME

    C->>F: WS MSG: END_GAME
    F->>R: SET room:{code}:status finished
    F->>R: PUBLISH room:{code}:events ROOM_STATE
    F-->>C: Broadcast ROOM_STATE { status: finished, seed, results[] }

2.4 房間狀態機

stateDiagram-v2
    [*] --> waiting : POST /api/rooms 建立

    waiting --> waiting : player_joined N<50
    waiting --> running : host START_GAME\nN>=2 AND 1<=W<=N-1\natomic: seed+ladderMap generated

    running --> revealing : host BEGIN_REVEAL\natomic: ladderMap+resultSlots written\nbcast ROOM_STATE_FULL status:revealing

    revealing --> revealing : REVEAL_NEXT (manual)\nor auto-timer fires\nrevealedCount < N

    revealing --> finished : host END_GAME\n(all paths revealed)\nbcast ROOM_STATE status:finished w/ seed\nTTL extend 1h

    finished --> waiting : host PLAY_AGAIN\nprune isOnline=false players\nbcast ROOM_STATE waiting

    waiting --> [*] : room TTL expired 4h
    finished --> [*] : room TTL expired 1h

3. 資料模型

核心 TypeScript 型別

export type RoomStatus = "waiting" | "running" | "revealing" | "finished";

export interface Player {
  readonly id: string;           // UUID v4
  readonly nickname: string;     // 1-20 chars, sanitized
  readonly colorIndex: number;   // 0-49
  readonly isHost: boolean;      // derived: id === room.hostId
  isOnline: boolean;
  readonly joinedAt: number;     // Unix ms
  result: 'win' | 'lose' | null; // null until revealed
}

export interface LadderSegment {
  readonly row: number;    // 0-indexed
  readonly col: number;    // col <-> col+1 has a rung
}

export interface LadderData {
  readonly seed: number;          // Mulberry32 seed = djb2(seedSource) >>> 0
  readonly seedSource: string;    // UUID v4 hex
  readonly rowCount: number;      // clamp(N*3, 20, 60)
  readonly colCount: number;      // = N
  readonly segments: readonly LadderSegment[];
}

export interface ResultSlot {
  readonly playerIndex: number;
  readonly playerId: string;
  readonly startCol: number;
  readonly endCol: number;
  readonly isWinner: boolean;
  readonly path: readonly PathStep[];
}

export interface Room {
  readonly code: string;
  title: string | null;
  status: RoomStatus;
  hostId: string;
  players: readonly Player[];
  winnerCount: number | null;
  ladder: LadderData | null;
  results: readonly ResultSlot[] | null;
  revealedCount: number;
  revealMode: "manual" | "auto";
  autoRevealIntervalSec: number | null;
  kickedPlayerIds: readonly string[];
  readonly createdAt: number;
  updatedAt: number;
}

5. WebSocket 事件設計

連線端點： WSS /ws?room={code}&token={sessionToken}

Server → Client 事件

事件類型說明
ROOM_STATE廣播房間狀態摘要（玩家加入/離線/踢出/中獎名額變更）
ROOM_STATE_FULLunicastWS 連線後給連線玩家（含 ladder + results + selfPlayerId）
REVEAL_INDEX廣播單一玩家路徑揭示（含 path、result）
REVEAL_ALL廣播全部剩餘路徑揭示（path 省略，64KB 限制）
PLAYER_KICKEDunicast推送給被踢玩家
SESSION_REPLACEDunicast同一 playerId 從新裝置登入，推送給舊連線
HOST_TRANSFERRED廣播主持人 60s 斷線後自動移交（Post-MVP）
PONGunicast回應客戶端 PING，含 RTT 計算用 ts
ERRORunicast操作失敗，含 code + message

Client → Server 訊息

訊息權限說明
START_GAMEHost開始遊戲（waiting 狀態）
BEGIN_REVEALHost開始揭曉（running 狀態）
REVEAL_NEXTHost手動揭示下一位（revealing 狀態）
REVEAL_ALL_TRIGGERHost一鍵全揭（revealing 狀態）
END_GAMEHost結束本局（revealing，所有路徑已揭曉）
PLAY_AGAINHost再玩一局（finished 狀態）
SET_REVEAL_MODEHost切換手動/自動揭示模式（revealing 狀態）
KICK_PLAYERHost踢除指定玩家（waiting 狀態）
UPDATE_TITLEHost更新房間名稱（waiting 狀態，0-50 chars）
UPDATE_WINNER_COUNTHost更新中獎名額（waiting 狀態，1 ≤ W ≤ N-1）
PINGAll應用層心跳 / RTT 量測

6. PRNG 設計

  • Mulberry32：32-bit 高品質 PRNG，純函式，零外部依賴
  • Seed 生成：seed = djb2(seedSource) >>> 0，seedSource 為 UUID v4
  • 洗牌：Fisher-Yates shuffle，以 Mulberry32 作為亂數來源
  • 共用程式碼：packages/shared/src/prng/，前後端完全一致
  • 安全性：seedSource 與 seed 在 running 狀態存於 Redis，不對外公開；finished 後可選擇性揭露供驗證

7. Redis 原子操作策略

操作機制說明
建立房間SETNX防重複建立，碰撞時重試最多 10 次
waiting → runningWATCH / MULTI / EXEC驗證 N、W 後設定 status=running
running → revealingLua Script原子寫入 seed、ladderMap、resultSlots、status=revealing
REVEAL_NEXTINCRrevealedCount 原子遞增，防重複計數
REVEAL_ALLSETrevealedCount = totalCount，原子設定
revealing → finishedMULTI / EXEC原子寫入 status=finished + extend TTL 1h

9. 部署架構

  • 容器：Distroless Node.js 20，多階段建構，非 root 執行
  • Kubernetes：Namespace ladder-room，Deployment minReplicas=2
  • HPA：依 ws_connections metric 自動擴縮（2-10 Pod），target=200 連線/Pod
  • Nginx Ingress：sticky session（cookie affinity），WebSocket upgrade 支援
  • Redis：StatefulSet 2 副本（master + replica），Service ClusterIP
  • Secrets：K8s Secret 存放 JWT_SECRET、REDIS_PASSWORD；ConfigMap 存放 PORT、LOG_LEVEL
