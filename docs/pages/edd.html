1. 系統概覽

Ladder Room Online 是一款基於 HTML5 Canvas 的多人線上爬樓梯抽獎系統，採用 WebSocket 長連接驅動即時遊戲狀態同步，支援最多 50 名玩家共享同一房間。後端以 Fastify 處理 HTTP REST 操作，ws 原生 WebSocket 處理即時通訊，Redis 同時承擔分散式狀態鎖（原子操作）、房間資料持久化與跨 Pod Pub/Sub 廣播的角色；前端以 Vanilla TypeScript + Vite 建構，透過 HTML5 Canvas 逐段繪製梯子揭示動畫，全程無任何 UI 框架依賴。

sequenceDiagram

    participant N as Nginx Ingress

    F->>R: SADD room:{code}:players {playerId}
    R-->>F: OK

    C->>N: GET WSS /ws?room={code}&token={token}
    N->>F: WebSocket Upgrade sticky session
    F->>F: Verify sessionToken HMAC-SHA256
    F->>R: HGET room:{code} load state
    R-->>F: Room JSON
    F-->>C: WS Connected
    F->>C: ROOM_STATE_FULL { room, players, ladder:null }

    Note over C,F: Host triggers START_GAME
    C->>F: WS MSG: START_GAME
    F->>R: WATCH room:{code} MULTI SET status=running EXEC
    R-->>F: EXEC OK
    F->>R: PUBLISH room:{code}:events ROOM_STATE
    F-->>C: Broadcast ROOM_STATE { status: running, rowCount }

    Note over C,F: Host triggers BEGIN_REVEAL
    C->>F: WS MSG: BEGIN_REVEAL
    F->>R: SET status=revealing (ladder generated atomically)
    F-->>C: Broadcast ROOM_STATE { status: revealing }

    C->>F: WS MSG: END_GAME
    F->>R: SET room:{code}:status finished
    F-->>C: Broadcast ROOM_STATE { status: finished, seed, results[] }

Server → Client 事件

5. PRNG 算法設計

// packages/shared/src/prng/mulberry32.ts

export function createMulberry32(seed: number) {
  return function () {
    let t = (seed += 0x6d2b79f5);
    t = Math.imul(t ^ (t >>> 15), t | 1);
    t ^= t + Math.imul(t ^ (t >>> 7), t | 61);
    return ((t ^ (t >>> 14)) >>> 0) / 4294967296;
  };
}

// packages/shared/src/prng/djb2.ts
export function djb2(str: string): number {
  let hash = 5381;
  for (let i = 0; i < str.length; i++) {
    hash = ((hash << 5) + hash + str.charCodeAt(i)) >>> 0;
  }
  return hash;
}

// rowCount formula
// rowCount = clamp(N * 3, 20, 60)
// N=3  → clamp(9,  20, 60) = 20
// N=10 → clamp(30, 20, 60) = 30
// N=21 → clamp(63, 20, 60) = 60

橫槓生成規則

  • 每 row 目標 max(1, round(N/4)) 條橫槓
  • 只連接相鄰欄位（|a - b| = 1）
  • 同 row 橫槓不重疊（|a - b| ≤ 1 視為衝突）
  • 每條橫槓最多嘗試 N × 10 次
  • 結果槽指派以 Fisher-Yates 洗牌確保 bijection（N 個起點對應 N 個唯一終點）

6. 安全設計

JWT Token 策略

  • 算法：HS256（jose 套件）
  • Payload：{ playerId, roomCode, role: "host" | "player", exp }
  • TTL：6 小時（21600 秒）
  • Host Token 僅在建立房間時發放；WS 升級與 host-only HTTP 操作均需驗證

防作弊機制

  • seed 在 status=finished 前禁止傳送給任何客戶端
  • WS 訊息大小上限 64KB，超過則拒絕並關閉連線
  • 被踢除玩家的 playerId 加入 kickedPlayerIds 黑名單（Redis Set）
  • 速率限制：60 msg/min per WsSession
