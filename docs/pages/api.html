API Design Document — Ladder Room Online

版本：v1.1  |  日期：2026-04-19  |  基於 EDD v1.3 + PRD v1.3

1. 概覽

Base URL 與版本策略

Production: https://api.ladder-room.online/api
Development: http://localhost:3000/api
WebSocket:   wss://api.ladder-room.online/ws
  • 所有 HTTP REST 端點掛載於 /api/。
  • WebSocket 不含版本路徑，由 query param token 的 JWT iat 隱含版本邊界。
  • 所有請求與回應均使用 Content-Type: application/json; charset=utf-8。
  • Host 專屬操作需在 Authorization Header 帶入 JWT：Authorization: Bearer <token>。

端點總覽

2. HTTP REST 端點

1. POST /api/rooms — 建立房間

建立新房間，創建者自動成為房主（host），並取得 host 身份的 token。

interface CreateRoomRequest {
  hostNickname: string;  // 1-20 個字元，禁止 null/控制字元
  winnerCount: number;   // 整數 >= 1
}

// Success Response — 201 Created
interface CreateRoomResponse {
  roomCode: string;  // 6 個字元，A-HJ-NP-Z2-9
  playerId: string;  // UUID v4，房主身份識別
  token: string;     // JWT HS256，role: "host"，TTL 6 小時
  room: Room;        // 完整房間物件
}

3. POST /api/rooms/:code/players — 加入房間

以暱稱加入指定房間，取得玩家 JWT。

interface JoinRoomRequest {
  nickname: string;  // 1-20 個 Unicode 字元
}

// Success Response — 201 Created
interface JoinRoomResponse {
  playerId: string;  // UUID v4
  token: string;     // JWT HS256，role: "player"，TTL 6 小時
  room: Room;
}
HTTP錯誤碼說明
404ROOM_NOT_FOUND房間不存在
409ROOM_FULL已達 50 人上限
409NICKNAME_TAKEN暱稱在此房間已被使用
409ROOM_NOT_ACCEPTING房間狀態非 waiting

4. DELETE /api/rooms/:code/players/:playerId — 踢出玩家

房主將指定玩家從房間移除。僅在 waiting 狀態有效。Success Response — 204 No Content

HTTP錯誤碼說明
401AUTH_INVALID_TOKENJWT 無效或格式錯誤
401AUTH_TOKEN_EXPIREDJWT 已過期
403PLAYER_NOT_HOST操作者不是房主
404PLAYER_NOT_FOUND目標玩家不存在此房間
409INVALID_STATE房間狀態非 waiting

5. POST /api/rooms/:code/game/start — 開始遊戲

房主觸發遊戲開始。後端生成 seedSource（UUID v4）並計算 rowCount，不生成梯子結構（梯子延遲至 BEGIN_REVEAL 時生成）。

注意：seed 與 seedSource 在 status=finished 之前均不對客戶端公開（PRD AC-H03-1, NFR-05）。梯子結構（LadderData）在 BEGIN_REVEAL 時才生成並寫入 Redis。

HTTP錯誤碼說明
400INSUFFICIENT_PLAYERS玩家數 N < 2
400PRIZES_NOT_SETwinnerCount 尚未設定（null）
400INVALID_PRIZES_COUNTwinnerCount >= N
403PLAYER_NOT_HOST操作者不是房主
409INVALID_STATE房間狀態非 waiting

6. POST /api/rooms/:code/game/reveal — 揭示結果

房主觸發結果揭示。mode: "next" 逐一揭示下一位玩家；mode: "all" 一次揭示全部剩餘。房間必須在 revealing 狀態。

interface RevealRequest {
  mode: "next" | "all";
}
HTTP錯誤碼說明
400INVALID_REVEAL_MODEmode 欄位不合法
403PLAYER_NOT_HOST操作者不是房主
409INVALID_STATE房間狀態非 revealing

7. POST /api/rooms/:code/game/end — 結束本局

房主在所有路徑揭示完畢後，手動觸發本局結束。房間從 revealing 轉為 finished，並廣播包含 seed 的 ROOM_STATE。

重要：REVEAL_ALL_TRIGGER 或最後一次 REVEAL_NEXT 本身不自動轉入 finished；必須由 Host 另外發送 END_GAME 才能完成狀態轉換。

HTTP錯誤碼說明
403PLAYER_NOT_HOST操作者不是房主
409INVALID_STATE房間狀態非 revealing 或路徑尚未全部揭示

8. POST /api/rooms/:code/game/play-again — 再玩一局

房主在遊戲結束後重置房間進行下一局。僅限 finished 狀態。離線玩家被移除，kickedPlayerIds 清空，winnerCount 若 >= 新玩家數則重置為 null。

HTTP錯誤碼說明
400INSUFFICIENT_ONLINE_PLAYERS在線玩家數 < 2
403PLAYER_NOT_HOST操作者不是房主
409INVALID_STATE房間狀態非 finished

9. GET /health — 健康檢查

{
  "status": "ok",
  "redis": "ok",
  "wsCount": 312,
  "uptime": 86423.7
}

10. GET /ready — 就緒檢查

回報後端是否已準備好接收流量（Redis 連線正常且無正在進行的啟動作業）。供 Kubernetes readiness probe 使用。

3. WebSocket 協議

連線建立

WSS wss://api.ladder-room.online/ws?room={code}&token={jwt}

連線成功後伺服器立即推送 ROOM_STATE_FULL 快照。

訊息格式

// Client → Server
interface ClientEnvelope {
  type: WsMsgType;
  payload?: Record<string, unknown>;
}

// Server → Client
interface ServerEnvelope {
  type: WsEventType;
  payload: Record<string, unknown>;
}

Server → Client 事件清單

事件觸發時機主要 Payload 欄位
ROOM_STATE狀態變更廣播room 物件（不含 ladder/seed）
ROOM_STATE_FULLBEGIN_REVEAL 後 / 重連room 物件含 LadderDataPublic（省略 seed）
REVEAL_INDEXREVEAL_NEXT 觸發playerIndex, path, result, revealedCount, totalCount
REVEAL_ALLREVEAL_ALL_TRIGGER所有剩餘玩家路徑
PLAYER_KICKEDKICK_PLAYER 後playerId
SESSION_REPLACED同 playerId 重複連線無
ERROR操作失敗code, message
