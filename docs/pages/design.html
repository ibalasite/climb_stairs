DESIGN — Client UI Spec

版本：v1.1  |  日期：2026-04-19  |  基於 PRD v1.3 · ARCH v1.2 · EDD v1.3 · API v1.1

Tech: Vanilla TypeScript + Vite · HTML5 Canvas · No UI framework

1. 設計方向

輕量遊戲感 × 現代簡約：以明亮、歡樂的色調配合清晰的幾何感，讓玩家在手機上一眼看懂當前狀態。乾淨的白底/淺底、強調色做點睛、動態揭曉是核心視覺高潮。

設計原則

  1. Mobile-first：320px 最小寬度，垂直滾動，單欄佈局優先
  2. 狀態即畫面：每個房間狀態（waiting / running / revealing / finished）對應獨一無二的 UI 組合
  3. Canvas 為主角：揭曉動畫佔主要可視區域，控制元素靠邊退讓
  4. 清晰的角色區分：主持人看到控制面板，玩家只看觀眾介面

3. 頁面清單

頁面URL Pattern適用角色房間狀態
首頁 / 入口/所有人—
建立房間/ → Modal主持人—
加入房間/?room=XXXXXX玩家—
等待大廳/room/:code主持人 + 玩家waiting
遊戲主畫面（梯子）/room/:code主持人 + 玩家running / revealing
結果頁面/room/:code主持人 + 玩家finished
被踢出/kicked被踢玩家—

頁面路由為 SPA，所有狀態切換在 /room/:code 內部依 WebSocket 事件動態渲染，不換 URL。

核心 UI 元件

  • RoomCodeInput：6 格獨立字元輸入（每格一個 <input maxlength="1">）或單一輸入框自動分割；自動轉大寫；透過 URL param ?room=XXXXXX 自動預填
  • NicknameInput：單行 <input>，1-20 字 Unicode 限制；即時 character count 提示（x / 20）
  • ActionButton（主要）：高度 48px（觸控最小目標）；全寬於 mobile；Loading spinner 狀態；Disabled 狀態

6. 遊戲主畫面（revealing）

Canvas 佈局

  • 頂部：玩家名稱標籤列（對應 N 條起始欄）
  • 中部：樓梯 Canvas（主要可視區域，佔螢幕 60-70%）
  • 底部：結果槽位（N 個格子，顯示中獎/未中獎符號）

Canvas 渲染規格

元素渲染方式顏色規則
垂直線（Vertical Lines）strokeRect / beginPath全體淡灰；自己的欄位強調色
橫槓（Rungs）beginPath / moveTo / lineTo與垂直線同色
行走動畫（Walking Animation）requestAnimationFrame 逐格推進路徑顏色：自己藍紫、中獎綠、未中獎紅、未揭曉灰虛線
結果槽（Result Slots）fillRect + text中獎槽綠底，未中獎槽紅底

主持人控制面板（revealing）

  • 手動模式：「下一位 →」按鈕（disabled 至全揭曉後）+ 「全部揭曉」按鈕
  • 自動模式：間隔設定（1-30s）+ 「停止」按鈕
  • 切換：手動 ↔ 自動模式切換按鈕
  • 結束：「結束本局」按鈕（所有路徑揭曉後才 enable）

8. 動畫規格

行走動畫（Walking Animation）

  • 驅動：requestAnimationFrame
  • 每個 step（row）動畫時長：~200ms（可隨 N 調整）
  • 遇橫槓時：先向左或右移動（150ms），再繼續向下
  • 目標 FPS：桌機 ≥ 30fps；手機 ≥ 24fps

已揭曉路徑呈現

  • 動畫結束後路徑固定顯示，結果槽更新
  • 其他未揭曉路徑以灰色虛線顯示

REVEAL_ALL 動畫

  • 接收到 REVEAL_ALL 廣播時，壓縮動畫時長（2 秒內完成全部渲染）
  • 若 2 秒內渲染未完成，直接跳至所有路徑的終止幀（final frame）

9. 響應式設計

斷點佈局調整
320px（最小）單欄，Canvas 縮放至螢幕寬度，控制元素垂直堆疊
768px（tablet）Canvas 可略大，控制面板可側邊顯示
1024px+（desktop）Canvas 佔主區域，控制面板固定側欄

Canvas 響應式縮放

// 響應式 Canvas 縮放：依父容器寬度計算欄位寬度
const colWidth = Math.floor(containerWidth / N);
const canvasWidth = colWidth * N;
const canvasHeight = Math.floor(canvasWidth * 1.5); // 高寬比 3:2

canvas.width = canvasWidth;
canvas.height = canvasHeight;
canvas.style.width = '100%';

10. 可及性（Accessibility）

  • 所有互動元素有 aria-label 或 aria-describedby
  • 色彩對比度：主文字與背景 ≥ 4.5:1（WCAG AA）
  • Canvas 結果同時以文字清單輔助呈現（螢幕閱讀器可讀）
  • 鍵盤可達：所有按鈕可透過 Tab 聚焦，Enter / Space 觸發
  • prefers-reduced-motion：使用者偏好減少動畫時，跳過行走動畫，直接顯示最終結果
