UI Design — 前端設計規格

1. 設計方向

風格：深色活動遊戲（Dark Party Game UI）。以深色背景搭配鮮豔的強調色，製造出如同夜間活動或直播的氛圍感。整體保持簡潔清晰，確保在投影機或螢幕分享下依然易讀。

核心設計原則

  • 清晰易讀：大型字體、高對比度，適合投影或直播展示
  • 即時反饋：所有 WebSocket 事件觸發即時 UI 更新，無需刷新頁面
  • 期待感：揭曉動畫製造緊張感，保留抽獎的儀式性
  • 零安裝：純瀏覽器，玩家 10 秒內可完成加入

3. 頁面清單

頁面路由對應狀態關鍵元件
Lobby（首頁）/—CreateRoomForm、JoinRoomForm
等待大廳/room/:codewaitingPlayerList、WinnerCountInput、StartGameButton、RoomCodeDisplay
遊戲主畫面/room/:coderunning / revealingLadderCanvas、HostControlPanel、StatusBar、RevealProgress
結果頁面/room/:codefinishedResultList、WinnerHighlight、SeedDisplay、PlayAgainButton

所有頁面為 SPA 單頁，依 WebSocket ROOM_STATE / ROOM_STATE_FULL 廣播自動切換視圖。

核心 UI 元件

  • ToastNotification：error / success，3.5s 自動消失，同時只顯示 1 條

核心 UI 元件規格

  • PlayerList：PlayerItem = 頭像占位符（首字母圓形）+ 暱稱 + 狀態 badge；Host 顯示 👑 badge；離線玩家：暱稱灰色 + ◌ 離線 chip；ROOM_STATE 事件觸發即時更新

  • StartGameButton：Disabled 條件：W 未設定 OR N < 2；顯示原因 tooltip

Canvas 規格（LadderCanvas）

屬性規格
最小寬度320px
高度min(80dvh, 600px)，響應式
縱欄數= N（玩家人數，2–50）
橫列數rowCount = clamp(N×3, 20, 60)
欄寬等寬分佈；最小欄寬 = 14px；N > 22 時 Canvas 橫向捲動
橫槓數max(1, round(N/4)) 條 / 列，粗 2px
路徑動畫requestAnimationFrame 驅動，目標 24fps（手機）/ 30fps（桌機）
座標系原點左上角；欄 i 中心 x = colWidth * i + colWidth / 2；列 j 的 y = rowHeight * j
running 狀態N 條灰色虛線（--color-path-hidden）佔位；無橫槓；等待 BEGIN_REVEAL 後重繪

路徑顏色語意

狀態顏色 Token線條樣式
未揭曉--color-path-hidden灰色虛線（dashed）
揭曉動畫中（當前玩家）accent 或 gold（自己）實線，寬 3px，帶光暈
揭曉完成，自己，中獎--color-success實線寬 3px
揭曉完成，自己，未中獎--color-danger實線寬 2px
揭曉完成，他人調色盤循環色（alpha 0.5）實線寬 1.5px
自己（未輪到）--color-accent虛線 dashed，寬 2px

揭曉動畫流程

REVEAL_INDEX 事件到達
  ↓
取得目標玩家 index 的 startColumn / path segments
  ↓
requestAnimationFrame loop 開始
  ↓
沿 segments 逐格前進（每格 duration 由 rowCount 決定）
  ↓
抵達底部 endColumn
  ↓
底部結果槽顯示 result（winner / loser）
  ↓
自己中獎 → 觸發 WinnerOverlay 動畫
  ↓
動畫結束 → 等待下一個 REVEAL_INDEX

8. 斷線 / 重連 UI

情境UI 行為
WS 斷線偵測顯示黃色 Toast「連線中斷，嘗試重新連線…」＋ spinner
重連成功（帶 playerId）Toast 消失；依 ROOM_STATE_FULL 恢復對應畫面
重連失敗 3 次後顯示「無法連線，請重新整理頁面」靜態提示
SESSION_REPLACEDToast「你的帳號已在其他頁面登入，此分頁已離線」；頁面凍結
running 狀態 15s 超時Toast「連線逾時，請重新整理頁面」（客戶端安全網）

9. 響應式設計

斷點行為
320px（最小）單欄版面；N ≤ 22 時 Canvas 適配容器；N > 22 時 Canvas 橫向捲動
768px等待大廳：玩家列表與設定區並排
1024px +遊戲主畫面：Canvas 左側，主持人控制面板右側固定欄

10. 無障礙（a11y）

  • 所有按鈕有 aria-label；disabled 狀態有 aria-disabled + tooltip
  • Canvas 附帶 aria-label 描述當前遊戲狀態（如「揭曉中，已完成 3/5」）
  • Toast 使用 role="alert" aria-live="assertive"
  • PlayerList 使用語意化 <ul> / <li>
  • 顏色對比度：主文字 / 背景 ≥ 4.5:1（WCAG AA）
  • prefers-reduced-motion：揭曉動畫退化為 0 動畫（即時顯示最終幀）

11. 動畫設計原則

  • 僅動畫 compositor-friendly 屬性：transform、opacity、clip-path
  • 避免動畫 layout 屬性：width、height、top、left
  • Canvas 路徑動畫：requestAnimationFrame，不用 CSS transition
  • REVEAL_ALL 模式：收到後 2 秒內若未完成，直接跳至最終靜態幀
  • 中獎 confetti：CSS keyframe + Canvas 粒子，使用 will-change: transform（動畫結束後移除）
